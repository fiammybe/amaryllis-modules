Event Manual

Keyfeatures

  • Create fast and simple an event and categorize them using different categories/colors
  • Categories are using the group permissions of ImpressCMS to allow different Groups to view categories and it's Events
  • Categories have a group permission for submitting new Events to handle Groups and different submit permissions
  • Import public calendars simple by using RSS-Feeds (e.g. public google calendars)

First Start

Preferences

The Module Preferences are short and simple:

  • Default View
    • The default view defines, which of the available views will be active, if you are calling the module. "Month" will display the current month, "Weekend" displays the current Week and "Day" the current Day
  • The first Day displayed in week view
    • Select, if the Monday should be the first day of a week or Sunday
  • Display Weekend
    • Select, the weekend need to be displayed or even not
  • Start time of the calendar
    • The start time means the time to be see at first. If the calendar is scaled (dependent on the theme you're using) you'll not see all hours. So the calendar will start at the predefined hour here. It will be possible to scroll up to the other hours
  • Min time for week/single day view.
    • Here you can change the minimum visible hour. E.g. set to 6 o'clock, all hours before won't be visible in the calendar. If you set an Event before this time, you just can see in month view
  • Max time for week/single day view.
    • The latest visible hour.
  • Slot minutes in Agenda view
    • The slot minutes to split an hour in week and day view
  • The following Date formats can be handled together:
    • Handle Date and Time formatting for calendars output. Form more informations check Date Format in Event Module
  • Use the UI Theme
    • As default, the calendar is styled by default ui theme included to core. To change the style you have two possibilities:
      • Search for another theme on jQuery UI Homepage (this will change all ui styles on the website or check out ui theme roller)
      • Turn UI Theme here off and design your custom design. This can also be done while ui-theme is enabled but it isn't that easies to design with ui theme in background

Fill up your calendar

The forms of the event module should all be small and simple enough. Just a few words around before you start up:

  • The Calendar Tab is ONLY necessary, if you like to render calendars via json feed, like Google Calendar. To read about Howto of adding Google Calendars go to Google Calendars in Event
  • The categories are used to define different colors/Text colors for Events to make it easier, to find the searched kind of Element. Google calendars are using the same feature. For Categories it is possible to set submit and view permissions. All allowed "view"-groups can view the events inside the category. The "submit"- Groups can submit events to the category
  • The Events have a field for contact person. As default, you'll find your account name and email inside their fields. If the contact name is another user and you type in the username, you'll get the user with avatar. Otherwise type in the Name and the E-Mail. If the email is registred to gravatar, you should see the avatar for this E-Mail. E-Mail Adresses are visible for registred users, not for guests.

Google Calendars in Event

The Event Module has the possibility to add public google calendars to your Website calendar. Before adding calendars, check the following:

  • You must first make your Google Calendar public or create a new calendar, which can made public
    • If you go to your google calendar, you'll find your calendars on the left side
    • Click to the Arrow on the right side next to the calendar you want to share and click on "Share Calendar"
    • Make sure "Share only my free/busy information" is unchecked. The calendar needs only to show data. Changings through event module or adding events is currently not possible.
    • Click next on calendar preferences (of the shared calendar) and check for the url. In the currently opened Setting box you'll find the Address of the calendar
    • Click on the Button to get the URL you need to add the calendar in your "Calendar" Tab in Event module.
    • Check out the Time-Zone Settings on the same Page. The time zone need to be set in your Calendar, when you're adding your Google Calendar. Examples for time-Zones are: "Europe/Berlin", "America/Chicago", "America/Santo_Domingo", "Europe/London" and a lot of more. Check out PHP.net Timezones, if you don't know the needed Timezone
      • There are a lot of useful public calendars you can use in gougle groups. For Example you might like to add Weeknumbers or Holiday.

      https://www.google.com/calendar/feeds/de.german%23holiday%40group.v.calendar.google.com/public/basic for (german) holidays

      https://www.google.com/calendar/feeds/e_2_de%23weeknum%40group.v.calendar.google.com/public/basic for week numbers

Date Format in Event Module

Now let's check the Date Preferences for Header and Column. At first let's get sure, we're talking about the same:

For Both cases use the following possibilities to adjust the Header and/or Column as you like it:
  • s - seconds
  • ss - seconds, 2 digits
  • m - minutes
  • mm - minutes, 2 digits
  • h - hours, 12-hour format
  • hh - hours, 12-hour format, 2 digits
  • H - hours, 24-hour format
  • HH - hours, 24-hour format, 2 digits
  • d - date number
  • dd - date number, 2 digits
  • ddd - date name, short
  • dddd - date name, full
  • M - month number
  • MM - month number, 2 digits
  • MMM - month name, short
  • MMMM - month name, full
  • yy - year, 2 digits
  • yyyy - year, 4 digits
  • t - 'a' or 'p'
  • tt - 'am' or 'pm'
  • T - 'A' or 'P'
  • TT - 'AM' or 'PM'
  • u - ISO8601 format
  • S - 'st', 'nd', 'rd', 'th' for the date

And some special Characters:

'...'
literal text
''
single quote (represented by two single quotes)
(...)
only displays format if one of the enclosed variables is non-zero

Module Blocks

Currently the module comes up with 2 Blocks:

  • Mini Calendar Block
    • The block is as simple as possible: it just shows a small month calendar. All Days are linked to the Agenda Day view, to make it as easy as possible to go from anywhere to a special day in Event calendar. The calendar has no more options to set (yet) and if you like to adjust it: The mini calendar is using the jQuery UI Datepicker, if you need a reference, how to adjust the calendar
  • Event List
    • This block shows just a simple Event List, linked to the calendar. You can select the category to be displayed and the time range. So for examle select no category and time range "current month" and you'll get a list with all events of the current month. The foloowing time ranges are available yet:
      • Current Day: Display all Events of the current Day
      • Day: Display all Events in 24hours
      • Current Week: Display all Events of the current Week
      • Week: Display all Events for one week
      • Current Month: Display all Events of the current Month
      • Month: Display all Events for one month

Templates

There're just a few templates available:

  • /event_admin.html
    • contains all necessary variables to display admin-side. Less informations enough, and I think, most people will never use this template for changes ;-)
  • /event_requirements.html
    • The same: only an admin template and will only be used, if the requirements check failed on installation.
  • /event_header.html & /event_footer.html
    • Header and footer holding additional informations like RSS-Feeds, Header informations, comment loop, notifications
  • /event_index.html
    • Currently displays the calendar and whatever needed for
  • /event_event.html
    • Dipsplaying full event Informations for a single event
  • /event_calconf.html
    • Is holding the calendar jquery configuration of the calendar
