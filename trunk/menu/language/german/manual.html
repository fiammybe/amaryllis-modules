Hauptfunktionen

  • Erstelle schnell menus mit multiplen links/sublinks
  • Links nutzen die Gruppen-Berechtigungen von ImpressCMS
  • Menus können mit Bildern/nur aus Bildern und mit diversem Text als Zusatzinfo erstellt werden (erfordert teilweise das erstellen eigener Menu-Skripte)

Erster Start

Erstelle das erste Menü

Das Erstellen neuer Menus ist mit das einfachste. Nur ein paar Anmerkungen pro Feld:

  • Titel und Beschreibung der Menus werden nicht angezeigt und dienen ausschließlich der besseren Identifizierung einzelner Menus
  • Die Art des Menus (horizontal/vertikal) wird nicht unbedingt genutzt. Derzeit ist es nur für das Megamenu ausschlaggebend. Alle anderen Menus haben die Scriptbezeichnung entsprechend gesetzt und richten sich danach. Wird die Unterscheidung für weitere scripte gebraucht wäre die smarty hierfür:

    <{if $block.menu_menu.kind == vertical}>

  • Anzeigen des Home-Links würde einen Link zu <{$icms_url}> automatisch zufügen, ohne extra einen Link erstellen zu müssen.
  • Text des Home link wird als Link-Text genutzt
  • Home image: Soll das Menu mit Bildern dargestellt werden (oder nur aus Bildern) Kann man hier das Bild für den Home-Link hochladen
  • Zeige Bilder: Hier kann man wählen, ob das Menu mit/ohne Bilder oder nur aus Bildern sein soll (letzteres hat derzeit kein vorgefertigtes Template).
  • Breite der Bilder: Gib die Breite der Hochgeladenen Bilder ein

    • Bilder werden derzeit ausschließlich in der hochgeladenen Größe gezeigt! Diese Funktion ist eher wichtig für die Abstände um die Bilder darzusstellen

  • Zeige die Beschreibung der Links oder nicht
  • ID des "<ul>"-tag. Um eines der vorgegebenen Menus zu nutzen benötignt man die folgenden ID's:
    • horiznav für eine css-basierte horizontale Navigation. Im Block "Simple Menu Block" nur das CSS-Template "horizontal.css" auswählen, keine js Datei erforderlich
    • accordion wird für beide accordion scripte genutzt. Im Block "Simple Menu Block" einfach das gewünschte Accordion Menu als css sowie js auswählen
    • sf-v-menu für ein vertikales superfish Menü. Im Block "Simple Menu Block" die superfish-vertical als css und js Datei auswählen
    • sf-h-menu für ein horizontales superfish Menü. In Block "Simple Menu Block" die superfish-horizontal als css und js Datei auswählen.
    • Um die Megamenüs zu nutzen ist keine ID benötigt. Sie haben ihre eigene ID und können mehrfach pro Seite eingesetzt werden

Das war's! Nach erfolgreicher Übermittlung sollte man jetzt direkt im Link-Item Formular landen und kann neue Links zu seinem Menu hinzufügen

Fülle das Menü mit Link-Items

  • Der Titel dient als Titel des Links
  • Die Beschreibung kann genutzt werden, um Links eindeutiger zu beschreiben oder, wenn es im Menü entsprechend gewählt wurde, die Beschreibung in den Links anzuzeigen
  • Menü: Wähle das zugehörige Menü
  • Url: Hier gibt es verschiedene Optionen:
    • {ICMS_URL} Wirft die Standard-ICMS-URL <{$icms_url}> aus
    • {MOD_URL} gibt die URL zum Modul Ordner. "http://example.de/modules/"
    • {UID} Gibt die User-ID des aktuellen nutzers aus
    • Im einfachen Accordion Menü (accordion_vertical.js/.css) haben alle Links ohne PID keine Funktion sondern dienen als Überschrift. Im zweiten Accordion brauchen nur Links ohne Sublink eine URL
  • target: _self oder _blank
  • Aktiv um einen Link zu daktivieren, so lange er nicht gebraucht wird
  • Gewichtung: Um Links nach Gewichtung Sortieren zu können
  • Betrachtungs-Berechtigung: Legt die Gruppen fest, die den Link sehen können

Das erste Menü wurde erstellt und mit Links aufgefüllt? Gehe zur Block-Administration

Modul Blöcke

Momentan kommt das Modul mit 2 Blöcken:

  • Simple Menü Block
    • The simple menu block is designed to display several menus

      • If you like to add your own script, just upload the css file and the js file to /modules/menu/scripts/ to make it possible to use the script within the module.

    • Use the following options to customize your menu:
      • Select Menu: Select the menu to be rendered
      • Order by: Order Menu items by title or weight
      • Sort: ASC or DESC
      • Select js file to include: Select the js file for the current menu
      • Select css file to include: Select the css file for the current menu

      • If you need special templates for your block, you can use the block id to seperate, which html should be displayed.

      <{if $block.block_id == YourBlockId}>

  • Megamenu Block
    • The MegaMenu Block displays the two possible Megamenus: "vertical" and "horizontal". Which of both is required to set in the theme. Several options makes it easy to adjust the block:
      • Select Menu: Select the menu to be rendered
      • Order by: Order Menu items by title or weight
      • Sort: ASC or DESC
      • Show images: Select yes, to display with images. A bit different to the Simple menu block..
      • Effect: Select the effect to be used
      • Event: Select the event to be used
      • Full width for submenus: Select, if the submenus should have the same width than first list items
      • Style: If you have selected an horizontal Menu, you can select several styles to get different menu colors.
      • Direction: Select the direction to be used for vertical menu. The direction is dependent on the side of the menu
  • All Blocks can be cloned to provide more menus

    • The module creates a new block position called "Menu Block". You can call the block position inside your theme by using "menu_block"

    <{foreach from=$xoBlocks.menu_block item=block}>
    <{include file="path_to_theme_folder/file_to_show_blocks.html"}>
    <{/foreach}>

Templates

There're just a few templates available:

  • /menu_admin.html
    • contains all necessary variables to display admin-side. Less informations enough, and I think, most people will never use this template for changes ;-)
  • /menu_requirements.html
    • The same: only an admin template and will only be used, if the requirements check failed on installation.
  • /menu_index.html
    • Not Necessary yet
  • /menu_single_item.html
    • Contains the link loop of a menu
