Keyfeatures

  • You can create multiple menus and several links/sublinks
  • Links are using icms view permission control
  • You might be able to display menus with images/only images and set a description to each link

First Start

Create your First Menu

Adding new menus should be most simple. Just a few things to know:

  • Title and description are only used to make the identification easier if you have multiple menus
  • Kind of Menu is not allways be used. Megamenus making use of the kind and others not, yet. If you need to decide in your scripts: The smarty would be

    <{if $block.menu_menu.kind == vertical}>

  • Displaying home link would mean to add a link to <{$icms_url}> automatically, without adding the link seperatly.
  • Text of the Home link will be the displayed text for home link
  • Home image: just if needed an image for mainlinks in your menu
  • Display images: select, if the images of your links should be displayed.
  • images display width will be necessary for several menus with images
  • Display item discription or not
  • Id of the ul tag is important for menus. If you like to use one of the predefined menus you should use the id for the menu:
    • horiznav for a css-based horizontal navigation. In Block "Simple Menu Block" just the CSS-Template "horizontal.css" needs to be selected, no js file required
    • accordion is used by both accordion scripts. In Block "Simple Menu Block" You can select one of the both js scripts and the same-named css file
    • sf-v-menu for a vertical superfish menu. In Block "Simple Menu Block" select superfish-vertical as css and js file
    • sf-h-menu for an horizontal superfish menu. In Block "Simple Menu Block" select superfish-horizontal as css and js file
    • To make use of the megamenu block no id is required. It makes use of it's own id

Done! After submitting you should directly be redirected to menu items to add your item for the menu

Fill menu with items

  • Title will be displayed as title of the link
  • Description can be used to make it easier to differentiate items or, if allowed in menu, the description can be displayed in links
  • Menu: Select the menu of the item
  • Url: There are several options:
    • {ICMS_URL} Will give you the default <{$icms_url}>
    • {MOD_URL} will give you the URL to the modules folder. "http://example.com/modules/"
    • {UID} will give you the user id of the current user
    • In simple Accordion Menu (accordion_vertical.js/.css) all links without parent will have no funktion, so just keep standard. In second Accordion only links without sublinks need a link
  • target: _self or _blank
  • Active to set a link to inactive, if not needed yet
  • Weight: to order by weight
  • View Permissions to set the granted Groups for the link. All selected groups will be able to see the link

You have created your Menu and filled up with some link-items? Go to blocks admin page and have a look for the Menu Block

Module Blocks

Currently the Module has two Blocks:

  • Simple Menu Block
    • The simple menu block is designed to display several menus

      • If you like to add your own script, just upload the css file and the js file to /modules/menu/scripts/ to make it possible to use the script within the module.

    • Use the following options to customize your menu:
      • Select Menu: Select the menu to be rendered
      • Order by: Order Menu items by title or weight
      • Sort: ASC or DESC
      • Select js file to include: Select the js file for the current menu
      • Select css file to include: Select the css file for the current menu

      • If you need special templates for your block, you can use the block id to seperate, which html should be displayed.

      <{if $block.block_id == YourBlockId}>

  • Megamenu Block
    • The MegaMenu Block displays the two possible Megamenus: "vertical" and "horizontal". Which of both is required to set in the theme. Several options makes it easy to adjust the block:
      • Select Menu: Select the menu to be rendered
      • Order by: Order Menu items by title or weight
      • Sort: ASC or DESC
      • Show images: Select yes, to display with images. A bit different to the Simple menu block..
      • Effect: Select the effect to be used
      • Event: Select the event to be used
      • Full width for submenus: Select, if the submenus should have the same width than first list items
      • Style: If you have selected an horizontal Menu, you can select several styles to get different menu colors.
      • Direction: Select the direction to be used for vertical menu. The direction is dependent on the side of the menu
  • All Blocks can be cloned to provide more menus

    • The module creates a new block position called "Menu Block". You can call the block position inside your theme by using "menu_block"

    <{foreach from=$xoBlocks.menu_block item=block}>
    <{include file="path_to_theme_folder/file_to_show_blocks.html"}>
    <{/foreach}>

Templates

There're just a few templates available:

  • /menu_admin.html
    • contains all necessary variables to display admin-side. Less informations enough, and I think, most people will never use this template for changes ;-)
  • /menu_requirements.html
    • The same: only an admin template and will only be used, if the requirements check failed on installation.
  • /menu_index.html
    • Not Necessary yet
  • /menu_single_item.html
    • Contains the link loop of a menu
