Keyfeatures

  • You can upload files to make them public your website or to share them with user groups.
  • You can categorize your files
  • Describe your file: key features, requirements, history, license, some screenshots and a lot more!
  • Bind "Downloads" to use with other modules. Currently:
    • "Album" Module to include image sets
    • "Catalogue" Module to sell your files
    • "Sprockets" Module for tag support

First Start

There are several things you should do before you try to upload a new file. Please, follow the instructions.

Allow the necessary File types

The Mime Types will be handled by the system, not by the module itself. Therefore we need to allow the needed mime types for the "Downloads"-Module and the groups, before we can upload files.

  • Go to System => Media => Mime Types. Here you will find the System Mime Type Handler.
  • Choose the Mime Types you want to allow for file upload. If you like to use file upload from front end beware of the group permissions per file. You need to select the Downloads Module to be allowed and the Groups, which you want to allow, too.
  • Remember the image upload, too. Actually the mime type permissions are used by the image upload, too. So please allow also jpeg, jpg, png and whatever you like to use.
  • Done! Now you can upload all file types you have allowed.

Check out the module preferences

There might be some preferences, you'd like to change before using the module first time. Go to Modules > Downloads > Preferences

Choose the preferences like preferred. Think, most preferences should be explained enough. So I'll list just a few here, which can be important to explain:

  • Group Permissions
    • The group permissions are only to set permissions for creating new categories in front end. All selected groups will have permissions to submit new categories.
  • Date Format
    • The Date format will be used for displaying the date throughout the module frontend. You need help to get the preferred date format? Look here for a detailed instruction.
  • Show Breadcrumb
    • Select "Yes" to display the breadcrumb in front end, "No" to disable the breadcrumb.
  • Show categories
    • Set the limit for categories to be listed on index view.
  • Select category columns
    • Select, how many columns you like to see on index view. The categories, limited by "Show categories", will be splitted into columns. 2 Columns are recommended for most themes, three might be possible, too. To use 4 columns it might be necessary to adjust css styles for your Theme.
  • Show Files
    • Set the limit of files to be listed on index view.
  • Screenshot Thumbnail width/Screenshot Thumbnail height
    • Set the dimensions of your image thumbnails for additional screenshots in px.
  • image upload width/image upload height
    • Set maximum upload dimensions in px
  • image file size/max file size
    • Set the maximum file size for uploads. Currently only the maximum file size is used for both.
  • How to display the file size?
    • Select the unit to display file size on single view. (bytes, kb(kilobytes), mb(megabytes) or gb(gigabytes) )
  • File image Thumbnail width/File image Thumbnail height
    • Set the display dimensions of file images in px
  • File limitations
    • The limitations are used in a drop-down select-field to Select the limitation for the current file. If you need different or additional limitations replace them or add more.
  • Show disclaimer, before an user can upload new files?
    • Select "Yes", if you like to show a disclaimer before the form for new file-uploads will be rendered.
  • Disclaimer for file upload
    • Please, DO NOT USE the predefined Disclaimer! Try to get a new one, which is more secured for your country. This one is ONLY AN EXAMPLE!!
  • Show disclaimer, before an user can download new files?
    • Select "Yes", if you like to show a disclaimer, before an user can download the file.
  • Disclaimer for file download
    • Please, DO NOT USE the predefined Disclaimer! Try to get a new one, which is more secured for your country. This one is ONLY AN EXAMPLE!!
  • file platform requirements
    • Requirements, which can be selected in a multi-select field while uploading a new file.
  • file license
    • Licenses, which can be selected in a multi-select-field while uploading a new file.
  • Use RSS-Feeds?
    • Select "Yes" to get a button with a RSS link.
  • Do you need version controll?
    • This option will show/hide several fields like file version, file history, version status (final, RC, beta etc)
  • Do you need related Files?
    • Select "Yes", if you need related files. You will get a list of files while you are uploading new files. There you can select related files.
  • Do you need File requirements?
    • Display/Hide the field for file requirements. Select "No" to hide the field from form.
  • Do you need file keyfeatures?
    • Display/Hide the field for file keyfeatures. Select "No" to hide the field from form.
  • Do you need a link to a Demo-Site
    • This option will display a field for a link to a demo website for each file.
  • Use 'Catalogue' Module?
    • You can use the Catalogue Module to link a single file to a single catalogue item, instead of a download link. If a file is linked to an item you do will get a link to the item and the price of the file from catalogue Module.
  • Use 'Sprockets' Module?
    • This will use the tag support of sprockets module. BEWARE: The module shows only items, which are marked as tag only, not as category or both!
  • Use 'Album' Module?
    • Select "Yes" to get a list of Albums. The selected Album and all contained images will be displayed on single file view to get more screenshots for a file.
  • Use mirror system?
    • Select yes, to get a new field for a mirror url to download your file from a mirror.
  • Do you need approvals for new mirrors?
    • Select "Yes", if you need approvals for new mirrors. If someone added a mirror while uploading a file from front end, the mirror will be set as denied and you will see it as an awaiting approval in system waiting block.
  • Do you need approvals for new files?
    • Select "Yes", if you like to approve new files before they are public. All new files will be marked as denied, if they are submitted from front end and you will see them as awaiting approvals in system waiting block.
  • Do you need approvals for new categories?
    • Select "Yes", if you like to approve new categories before they are public. All new categories will be marked as denied, if they are submitted from front end and you will see them as awaiting approvals in system waiting block.
  • Do you like to allow guests to vote?
    • If you like to allow guests to vote your file, select "Yes". Otherwise they will only get a popup to register/login or cancel the rating.
  • show reviews?
    • Select "Yes" to make reviews public in front end. They will be displayed in a new tab in single file view.
  • do you like to allow guests to submit reviews?
    • Select "Yes", if you like to allow guests to submit file reviews. Otherwise they will only get a link to register/login or cancel.
  • show reviews email?
    • Select "Yes" to display E-Mail of review-submitters. (Not recommanded)
  • How to display the email-address in front end?
    • Select the display method of submitters email. Spam protected: email [at] example dot com, usual: email@example.com. If you like to use spam protection against banned list.
  • show user avatar in reviews?
    • Select "Yes" to display the user avatar in reviews.
  • How many reviews should be shown in one page?
    • Set limit of reviews before rendering page navigation
  • Order reviews date
    • Order reviews by published date DESC/ASC
  • How many downloads of one file before it's popular
    • Select the amount of downloads before a file is popular and will be displayed in "Most popular" Block. Select "0" to turn off.
  • How many days to provide one file as new
    • Enter the days, a file will be marked as new after submitting. Enter "0" to turn off.
  • How many days to provide one file as updated after editing
    • Enter the days, a file will be marked as updated after updating. Enter "0" to turn off.
  • All other preferences are only effected for system comment rules and notifications.

Done! Next step.

Adjust index page

Now Let's have a look at the index Page. For this find the tab "Edit Indexpage" in Module Menu. Select the Index image, which you'd like to display on index or upload a new one using the image upload field. The Index image will be displayed on all module sites.

The Index header is just a header on module home, like the heading. It can be used for some more Text on module home to explain/invite guests to register and upload or whatever you like to.

The index footer is like the index image displayed on all sites of the module.

Create a category

The next most important step would be creating a category. Otherwise you can not see the file upload page. Go to Tab 'Manage categories' > Add Category. Just Fill out the fields. You can select an image for the category or upload a new one. You also can upload a set of new category images to /uploads/downloads/categoryimages/. The module will read the image files inside the folder and show them inside the select box. Done.

Upload Files

Finally you can start uploading files. Go to Tab 'Manage Files' > 'Add File' and fill out the fields. Done! Now you should see your first category and file inside the frontend.

Reviews and Log

The review system is just to add reviews in frontend. You can edit them, if necessary in module ACP, but you will not be allowed to add new one. Please, use frontend to add reviews. Also the reviews doesn't have a real permission controll. You may disable adding for guests, so they will get a popup-message to register or login for submitting a review.

For Log System it's a bit more strange: You cannot add Log items and you'll be not allowed to change fields. It's just for logging and statistics. If you have an spammer problem you can get the IP to bann the sammer from your site.

Blocks

Currently you have 5 blocks for 'Downloads' module:

  • Recent files
    • The "Recent Files" block will display the recently uploaded files and sort them by published date, DESC.
  • Recent updated files
    • This block will display recently updated files and sort them by updated date, DESC.
  • Most popular files
    • This block content will be dependent from your preferences. Only files, which where called more about the selected number will be shown inside the block.
  • Category Menu block
    • Here you can see the category like a menu. You also can select one special category to display all subcategories of this one.
  • Gallery Block
    • The Gallery Block will display recent uploaded files. The Block requires a file image. If the Block doesn't has a file image, the file won't be displayed.

You can clone all blocks, if needed.

"Downloads" Templates

The Main Templates:

  • downloads_admin.html
    • Contains all Template for ACP. Rendering the tables and forms and the manual.
  • downloads_category.html
    • Template for a single category/subcategory within a loop on category index view.
  • downloads_download.html
    • Contains a single download within a loop on index view.
  • downloads_footer.html
    • Footer included in all frontend Files. Contains Admin-link Comment rules and notification container
  • downloads_forms.html
    • Contains the forms for adding/deleting new categories/downloads on front end
  • downloads_header.html
    • Header file included in all front end templates. Contains breadcrumb, RSS-Link and Index-Image.
  • downloads_index.html
    • The Main template, which lists all loops.
  • downloads_requirements.html
    • Only used, if module requirements check failed.
  • downloads_singledownload.html
    • Displays a single file on single file view.
  • downloads_singlereview.html
    • Displays a single review in review loop on single file view.

Import from Wf-Downloads

The module contains an import script for Wf-Downloads. A few things, before you start importing:

  • If you're updating an older site and you also like to use article and import from news(old)/smartsection, make sure, you first use the import of article before importing wfd data.
  • Make sure, it's a clean "Downloads" installation. It might be difficult if you have created categories/downloads first.
  • You will loose your old voting and metainformation data

Ok, let's start. It's easy enough.

  • At first: you will need to uninstall Wf-Downloads before importing. Make a backup of ALL WF Downloads tables first.
  • After successfully updated system you can install "Downloads". Don't forget now to import your sql file.
  • Importer will find the tables itself and present you the buttons. Start with first button and click them step by step.
  • Don't worry, if the waiting time is a bit longer. For an import of about 4000 Entries in another module I needed to wait nearly 2 minutes!
  • Check, if all categories/files are present and there are no mistakkes.
